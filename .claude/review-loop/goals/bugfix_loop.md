目的:
指定された不具合を Claude が修正し、Codex reviewer と judge を通しながら反復で収束させる。

対象:
今回の不具合に直接関係するファイルだけを変更する。
無関係なリファクタはしない。

やること:
不具合の原因を特定する。
最小変更で修正する。
必要ならテストを追加または更新する。
各ラウンド後に reviewer と judge の結果を反映する。

禁止事項:
症状と無関係な設計変更を入れない。
Codex reviewer / judge 側に編集・テスト実行・build 実行をさせない。

完了条件:
不具合が再現しなくなる。
関連テストが通る、または未実行理由が明確に残る。
judge が `done` を返す。
