目的:
指定された機能を Claude が実装し、Codex reviewer と judge を通しながら仕上げる。

対象:
要求実現に必要なファイルだけを変更する。
必要なら README や関連テストも更新してよい。

やること:
機能要件を満たす実装を行う。
変更内容を小さな単位でまとめる。
各ラウンド後に reviewer と judge の結果を見て次の行動を決める。

禁止事項:
要求にない大規模リファクタを混ぜない。
Codex reviewer / judge 側に編集・テスト実行・build 実行をさせない。

完了条件:
要求機能が実装される。
関連テストまたは確認手順が揃う。
judge が `done` を返す。
