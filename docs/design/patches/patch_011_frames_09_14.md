# Patch 011 — フレーム 09〜14 ピクセル定義

## 背景

設計書 v11 §4 では frame 09〜14 について「完了くるくるアニメ / エラー上下シェイク」と用途を記載しているが、各フレームのピクセル座標定義がない。元の参照 PNG（`charactor--09.png` 〜 `charactor--14.png`）もリポジトリに存在しないため、仕様ギャップとして本 patch で定義する。

## 設計方針

- 新しい描画プリミティブを追加しない
- 既存の顔・目ソケット・瞳・×マークを組み合わせる
- §8 のルール（フレーム丸ごと切り替え、座標計算禁止）を遵守

## DONE フレーム（09, 12, 13, 14）

顔色: `faceDone (#C08888)` 固定。目ソケット・瞳サイズは通常と同じ（ソケット 4×8、瞳 2×6）。
瞳位置を時計回りに循環させ「くるくる」を表現する。

| フレーム | 瞳位置 | 座標（左/右） | GazeFrame 対応 |
|----------|--------|---------------|----------------|
| 09 | 右上 | 左(7,3) 右(15,3) | f05_rightUp |
| 12 | 右下 | 左(7,5) 右(15,5) | f02_rightDown |
| 13 | 左下 | 左(5,5) 右(13,5) | f03_leftDown |
| 14 | 左上 | 左(5,3) 右(13,3) | f04_leftUp |

### DONE アニメーション順

```
08(中央) → 09(右上↗) → 12(右下↘) → 13(左下↙) → 14(左上↖) → 13(左下↙) → 12(右下↘)
```

各ステップ間隔: 120ms。合計約 720ms。
最終フレーム 12 は f02_rightDown = idle と同じ瞳位置のため、done→idle 遷移がスムーズ。

## ERROR フレーム（10, 11）

顔色: `faceError (#C06868)` 固定。×マーク描画は frame 07 と同一。
顔全体の Y 座標を ±1dot オフセットし「上下シェイク」を表現する。

| フレーム | Y オフセット | 顔位置 | 効果 |
|----------|-------------|--------|------|
| 10 | -1 dot | (3, 0) | 顔が上端にぴったり |
| 11 | +1 dot | (3, 2) | 顔が下端にぴったり |

### ERROR アニメーション順

```
07(通常×) → 10(×が↑) → 11(×が↓) → 10(×が↑) → 07(通常×)
```

各ステップ間隔: 80ms。合計約 320ms。
