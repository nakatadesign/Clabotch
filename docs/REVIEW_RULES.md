# REVIEW_RULES.md — レビュールール・禁止事項

## 絶対禁止事項

- ファイルの**編集・作成・削除・書き込み**は一切禁止
- コードの読み取り・分析・評価は許可（むしろ必須）

## タイムアウト防止ルール（必須）

- `xcodebuild`・`swift build`・`swift test` を実行しない
- 1回のレビューは変更ファイル3つ以内に絞る
- コードの読み取りと分析のみで完結させる
- テスト結果はClaude Codeから提供された情報を使用する

## 計画レビューのルール

- 実装計画のレビュー依頼時はテキスト分析のみ行う
- 技術的リスク・抜け漏れ・改善提案を日本語で指摘する
- 実装・ファイル操作は一切しない

## 許可される操作

- コードファイルの読み取り（Read / Grep）
- 分析・評価・レビューコメントのテキスト出力
- 改善提案を言葉で伝える
- `docs/ARCHITECTURE.md` に基づくアーキテクチャ適合性の判定

---

## レビュー観点（必ず全項目評価）

### 1. セキュリティ

- Unix domain socket のパーミッション
- stdin JSON の検証・バリデーション
- jq 必須化・session_id 欠損時の drop-and-log

### 2. パフォーマンス

- メインスレッドのブロッキング
- タイマーリーク・メモリリーク
- GazeController の polling 間隔

### 3. 保守性

- 命名規則（Swift API Design Guidelines）
- 関数の責務分離
- MascotPhase の状態遷移の明確さ

### 4. 設計・アーキテクチャ

- スレッド境界ルール（ARCHITECTURE.md 参照）
- StateMachine / GazeController の所有権
- EventDeduplicator のグローバル単一インスタンス

### 5. 型安全性

- Swift 型定義の適切さ
- `Any` / `AnyObject` の不必要な使用
- enum の網羅性（switch exhaustiveness）

### 6. テスタビリティ

- pure function の分離（EventParser）
- 副作用の分離
- Mock可能な設計か
